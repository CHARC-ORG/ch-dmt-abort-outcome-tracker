## What this is

This is a **research data collection instrument**, distinct from the companion 
[educational dose estimator](https://github.com/CHARC-ORG/nndmt-ch-dose-estimator).

Where the dose estimator helps anyone understand their device setup, this tracker
is built to do one thing: collect structured, longitudinal data on real-world nnDMT
abort attempts from community participants — the kind of data a clinical trial team
would need to design a protocol grounded in how people actually use this medicine.

---

## The research gap

Dr. Schindler's Yale/VA DMT survey (Neurology, 2025) established that 67.5% of
community DMT users report complete attack elimination. That's the efficacy signal.
What the literature still lacks entirely:

- What estimated doses (derived from device parameters) correlate with complete abort?
- Does onset-to-first-inhale timing predict outcome?
- What Kip level at time of use affects success rate?
- What physical and psychological adverse effects occur, at what doses, under what conditions?
- Are non-responders failing pharmacologically — or are they inadvertently combusting their DMT above 220°C?
- What does within-person dose-response look like across repeated attempts?

This tracker captures all of these variables, per session, per participant, building
a longitudinal dataset that no published study has produced for DMT CH abort.

## Variables captured per session

| Field | Detail |
|---|---|
| **Device + dose model** | Full 7-variable pharmacokinetic calculation |
| **Est. bioavailable DMT (mg)** | From device type, coil, voltage, concentration, technique |
| **Estimated temperature (°C)** | Watt→temp model; TC/induction devices use set temp directly |
| **DMT combustion flag** | Automatic warning when setup exceeds 220°C threshold |
| **Onset → first inhale** | Shadow through peak — 10 time range options |
| **Kip at first inhale** | 1–10 with full clinical descriptors |
| **Kip at attack peak** | 1–10 — may differ significantly from vape timing |
| **Abort outcome** | 7 tiers: complete / significant / partial / none / worsened / too intense / adverse |
| **Time to pain change** | Under 30 sec through delayed / no effect |
| **Psychedelic intensity** | 0–10 with live descriptors |
| **Sitter presence** | Designated sitter / nearby / alone |
| **Physical side effects** | 14 options, multi-select |
| **Psychological side effects** | 13 options, multi-select — with positive lasting effects as distinct category |
| **Open notes** | Free text — adverse experiences especially captured |

## Participant ID system

Three-word passphrase (e.g. `amber-orbit-seven`) — generated by the tool or user-defined.

- Stored only in the participant's browser
- **Never transmitted in identifiable form**
- One-way SHA-256 hashed before contributing to community pool (not reversible)
- Links sessions across visits without any identifying information
- Participants can export or delete their data at any time

## Dose model

```
mg_bioavailable =
  (vol_ml × conc_mg_ml) ÷ puffs_per_cart × puffs_taken
  × temperature_efficiency
  × carrier_factor × crystallization_factor
  × (puff_duration × inhalation_style)
  × hold_retention × puff_interval

Temperature model (ceramic/mesh coils):
  T_coil (°C) ≈ 80 + (W × 4.5)
  W = V² ÷ R (Ohm's Law)

TC/induction devices use set temperature directly.
E-mesh uses loaded mass directly (no liquid calculation).
```

**Uncertainty:** ±18% (e-mesh) to ±40% (unknown/prefilled carts).
Individual estimates are ranges, not precise doses.

**DMT combustion threshold:** ~220°C. The tracker automatically flags when a
device setup is estimated to exceed this, as combustion users may be reporting
"non-response" that is actually a technique/device failure.

## Validation status

**Current status:** Limited community launch — ClusterBuds Discord, July 2025.

Feedback actively sought on:
- Device list completeness and accuracy
- Dose model agreement with user experience
- Outcome and side effect category coverage
- Kip descriptor accuracy for self-rating
- Onset timing option granularity

**Submit issues** via GitHub Issues with the `validation` label.

**Known limitations:**
- Dose model ±30–40% uncertainty for liquid-based methods
- Self-reported Kip has inherent subjectivity
- Community pool uses browser storage (not a server database) in v1.0
- Requires `window.storage` API (Claude.ai artifact) or localStorage fallback
